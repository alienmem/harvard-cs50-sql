# Week 1: Relating

## Topics Covered

- **Relational Databases** — Understanding relationships between tables
- **Relationships** — One-to-one, one-to-many, many-to-many
- **Entity Relationship Diagrams (ER Diagrams)** — Visualizing database design with crow's foot notation
- **Keys** — Primary keys (unique identifiers) and foreign keys (relationships)
- **Subqueries** — Nested SELECT statements for complex queries
- **IN Operator** — Set membership testing
- **JOINs** — Combining data from multiple tables
  - **INNER JOIN** — Matching rows from both tables
  - **LEFT JOIN** — All rows from left table plus matches
  - **RIGHT JOIN** — All rows from right table plus matches
  - **FULL JOIN** — All rows from both tables
  - **NATURAL JOIN** — Automatic joining on same column names
- **Set Operations**
  - **INTERSECT** — Common rows between queries
  - **UNION** — Combined results from multiple queries
  - **EXCEPT** — Rows in first query but not second
- **GROUP BY** — Grouping rows for aggregation
- **HAVING** — Filtering aggregated results
